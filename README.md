# EcSops

Welcome to the **EcSops** repository! This project contains a collection of Python scripts, automated Standard Operating Procedures (SOPs), and SQL queries used for managing operational workflows, lead transfers, customer service interactions, and data reporting.

---

## 📁 Project Structure

This project is organized into sub-domains to keep code separated by functionality:

```text
e:\EcSops\
├── .env                              # Stores sensitive environment variables (e.g., DB credentials, API keys)
├── config.py                         # Global configurations and constants
├── requirements.txt                  # Python dependencies required to run the scripts
├── README.md                         # Project documentation and setup instructions
├── .vscode/                          # VSCode settings
├── data/                             # Directory for local CSV data processing
│   ├── input/                        # Place raw source files here for scripts to consume
│   └── output/                       # Output generated by extraction and report scripts
├── logs/                             # Execution logs generated by running the automated scripts
├── sql/                              # SQL Views, Queries, and Reporting Scripts
│   ├── engagement_report.sql
│   ├── sales_bible_vw_query.sql
│   └── sales_productivity_view.sql
├── src/                              # Main Python source code directory
│   ├── db/                           # Database connection logic
│   ├── leads/                        # Lead management and contact transfer scripts
│   ├── customer_service/             # Customer service allocation and operations
│   ├── payments/                     # Payment approvals and offline payment updates
│   └── utils/                        # Maintenance, recovery, and general utility SOPs
└── tests/                            # Integration and Unit tests
```

## 🚀 Setup & Installation

Follow these steps to set up the project on your local machine:

### 1. Prerequisites
- Python 3.9+
- Access to the target databases (validate credentials with your team)

### 2. Create a Virtual Environment (Optional but Recommended)
```bash
python -m venv venv
```
Activate the virtual environment:
- **Windows**: `venv\Scripts\activate`
- **Mac/Linux**: `source venv/bin/activate`

### 3. Install Dependencies
Run the following command to install required Python libraries:
```bash
pip install -r requirements.txt
```

### 4. Configure Environment Variables
You need an `.env` file in the root directory to store sensitive credentials safely. It should never be committed to version control.
Example `.env`:
```env
DB_HOST=your_database_host
DB_PORT=5432
DB_USER=your_db_username
DB_PASSWORD=your_secure_password
```

## 🛠️ Key Workflows & Scripts

### Lead Management (`src/leads/`)
Scripts handling the creation, pulling back, and transferring of leads across platforms:
- `Pullback_lead_creation.py`
- `custom_lead_contact_transfer.py`
- `lead_contact_transfer_csv_based.py`
- `transfer_leads.py`

### Customer Service (`src/customer_service/`)
Routines for handling data extracts and mapping for customer service entities (Retail, Dhanwaan, etc.):
- `Retail_customer_service.py`
- `dhanwaan_customer_service.py`
- `customer_service data extract v2.py`

### Offline Payments (`src/payments/`)
Handles the processing and reconciliation of offline payment updates:
- `approve_offline_payments.py`
- `offline_payment_data_update.py`
- `winback_data_update.py`

### SQL Reporting (`sql/`)
Key reporting views needed for dashboards and business intelligence:
- `engagement_report.sql`: Daily engagement metrics.
- `sales_productivity_view.sql`: Base view for sales productivity dashboards.

## 🚨 Best Practices

1. **Never Commit Sensitive Information**: Ensure passwords, tokens, and personally identifiable information (PII) are excluded from the repository.
2. **Virtual Environments**: Always run your scripts inside an active virtual environment to avoid dependency conflicts.
3. **Log Everything**: If you are creating a new script, ensure you use the python `logging` module and write logs to the `logs/` directory for easier debugging.

## 🤝 Need Help?
For issues with database access or script behavior, please consult `config.py` for standard configurations or check the corresponding `.env` setup.
